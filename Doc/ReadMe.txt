作者：亢琦
项目代号：Mercury
运行环境：罗克佳华MSP430蓝板
软件功能：
	1、通过DTU接收平台下发的212报文，根据报文内容判断数采仪当前在线状态。
	2、根据数采仪在线状态控制DO输出，实现报警功能。
	3、实现了历史数据保存功能。
	4、通过屏幕可以查询最近50条历史数据。

系统架构:
	N个数采仪-->数据中心平台-->监箱(本程序运行平台)
	
系统架构说明:
	本软件为监箱程序，监箱上有与数采仪一一对应的报警指示灯，平台检测数采仪的上下线情况
	并将信息发送至监箱，监箱根据信息内容控制对应报警指示灯的开关。
	
212协议通讯过程说明：
	1、数据中心平台与监箱之间采用212协议通讯。由于212协议规定的内容不能满足平台与监箱的通讯需求，
	故由	双方共同约定使用特定的命令字及字段完成对数采仪状态的监控需求。
	2、监箱启动后会想平台定期发送心跳包。
	3、平台定期及在监控的数采仪状态发生改变时会向监箱发送状态报文。
	
心跳报文:
	##0102ST=32;CN=9021;MN=03510210100151;PW=123456;QN=20140808120000000;CP=&&&&4400
状态报文:
	##0117ST=32;CN=9019;MN=03510210100151;PW=123456;CP=&&DataTime=20140808120000;
	EN=03510210100152,ES=O,EO=Y;EN=03510210100153,ES=D,EO=N;EN=03510210100154,ES=O,EO=Y&&3180
	
命令字及字段说明：
	1、命令字9021: 用于监箱向平台发送心跳包;
	2、命令字9019: 用于平台向监箱通知当前数采仪状态;
	3、字段EN: 设备号(Equipment Number)，取值为当前所监控的数采仪的MN号；
	4、字段ES：设备状态(Equipment Status)，取值为O(Online)或D(Dropline）；
	4、字段EO：设备状态(Equipment OverStatus)，取值为Y(Yes)或N(No);
